# Nix Flake DevShell Skill

Create a reproducible development environment using Nix flakes with flake-parts and the dendritic cells pattern.

## Overview

This skill creates a devShell that provides:
- **fish** shell with **starship** prompt
- **zoxide** for fuzzy directory jumping
- **eza**, **ripgrep**, **fzf** for navigation
- **neovim**, **git** for development
- **direnv** auto-activation
- Fish abbreviations for common commands

## Pattern: Dendritic Cells

```
project/
├── flake.nix                    # Root: inputs, cell loading
├── flake.lock                   # Pinned dependencies
├── nix/
│   └── cells/
│       └── devshells/
│           ├── default.nix      # DevShell definition
│           └── config/          # Shell configs
│               ├── starship.toml
│               └── config.fish
└── .envrc                       # direnv entry point
```

## When to Use

- Starting a new Nix-based project
- Want reproducible dev environments across machines
- Prefer fish shell with modern tooling
- Using flake-parts for modularity

## Quick Start

### 1. Create Root flake.nix

```nix
{
  description = "PROJECT-NAME - description";

  inputs = {
    nixpkgs.url = "github:nixos/nixpkgs/nixos-unstable";
    flake-parts.url = "github:hercules-ci/flake-parts";
  };

  outputs = inputs@{ flake-parts, ... }:
    flake-parts.lib.mkFlake { inherit inputs; } {
      systems = [ "x86_64-linux" "aarch64-linux" "aarch64-darwin" ];

      imports = [
        ./nix/cells/devshells
      ];
    };
}
```

### 2. Create Directory Structure

```bash
mkdir -p nix/cells/devshells/config
```

### 3. Create DevShell Cell

File: `nix/cells/devshells/default.nix`

```nix
{ inputs, ... }:
{
  perSystem = { config, self', inputs', pkgs, system, ... }:
    {
      devShells.default = pkgs.mkShell {
        name = "PROJECT-NAME";

        packages = with pkgs; [
          # Core shell
          fish
          starship
          direnv
          nix-direnv

          # Navigation & search
          eza
          ripgrep
          zoxide
          fzf

          # Editor & dev tools
          neovim
          git

          # LSPs & formatters
          nil
        ];

        shellHook = ''
          echo "Entering PROJECT-NAME devShell..."
          export SHELL=${pkgs.fish}/bin/fish

          # Set starship config
          export STARSHIP_CONFIG=${./config/starship.toml}

          # Set fish config directory for this shell
          export XDG_CONFIG_HOME="$PWD/.nix-devshell-config"
          mkdir -p "$XDG_CONFIG_HOME/fish"
          cp ${./config/config.fish} "$XDG_CONFIG_HOME/fish/config.fish"

          # Start fish if not already in fish AND running interactively
          # CRITICAL: [[ -t 0 ]] check required for 'nix develop --command' to work
          if [[ -z "$FISH_VERSION" && -t 0 ]]; then
            exec ${pkgs.fish}/bin/fish
          fi
        '';
      };
    };
}
```

### 4. Create Starship Config

File: `nix/cells/devshells/config/starship.toml`

```toml
# Minimal starship config
format = """
$directory$git_branch$git_status
$character"""

[directory]
truncation_length = 3
truncation_symbol = ".../"
format = "[$path]($style) "
style = "cyan"

[git_branch]
format = "[$branch]($style) "
style = "purple"

[git_status]
format = '([$all_status$ahead_behind]($style) )'
style = "yellow"

[character]
success_symbol = "[>](green)"
error_symbol = "[>](red)"

[username]
disabled = true
[hostname]
disabled = true
[time]
disabled = true
```

### 5. Create Fish Config

File: `nix/cells/devshells/config/config.fish`

```fish
# Initialize tools
starship init fish | source
zoxide init fish | source
direnv hook fish | source

# Git abbreviations
abbr -a g git
abbr -a gs 'git status'
abbr -a gd 'git diff'
abbr -a ga 'git add'
abbr -a gc 'git commit'
abbr -a gl 'git log --oneline -15'

# Navigation abbreviations
abbr -a l 'eza --icons --group-directories-first'
abbr -a ll 'eza -la --icons --group-directories-first'
abbr -a lt 'eza --tree --icons'

# Editor abbreviations
abbr -a n nvim

# Welcome message
echo "Welcome to PROJECT-NAME devShell"
```

### 6. Create .envrc

```bash
# Auto-activate devShell when entering project
use flake
```

### 7. Create .gitignore

```
# Direnv
.direnv/

# Fish config generated by devShell
.nix-devshell-config/

# Nix build results
result
result-*
```

### 8. Lock Inputs

```bash
nix flake update
```

## Critical Implementation Notes

### Interactive Terminal Check (REQUIRED)

**The shellHook MUST include `[[ -t 0 ]]` check:**

```bash
# WRONG - breaks 'nix develop --command'
if [[ -z "$FISH_VERSION" ]]; then
  exec fish
fi

# CORRECT - allows non-interactive usage
if [[ -z "$FISH_VERSION" && -t 0 ]]; then
  exec fish
fi
```

Without this check, `nix develop --command` fails because fish unconditionally replaces the shell process.

### Package Availability

- `determinate-nix` does NOT exist in nixpkgs
- Use standard `nix` package instead
- If Determinate Nix features needed, install separately

## Customization

### Adding Packages

Add to the `packages` list in `default.nix`:

```nix
packages = with pkgs; [
  # ... existing packages ...
  
  # Add your tools
  nodejs
  python3
  rustc
  cargo
];
```

### Adding Abbreviations

Add to `config.fish`:

```fish
abbr -a your-abbr 'your-command'
```

### External Flake Inputs

To add external tools like `llm-agents`:

```nix
# In flake.nix inputs:
llm-agents = {
  url = "github:numtide/llm-agents.nix";
  inputs.nixpkgs.follows = "nixpkgs";
};

# In devshells/default.nix:
{ inputs, ... }:
{
  perSystem = { config, self', inputs', pkgs, system, ... }:
    let
      llm-agents-packages = inputs.llm-agents.packages.${system};
    in
    {
      devShells.default = pkgs.mkShell {
        packages = with pkgs; [
          # ... other packages ...
          llm-agents-packages.pi
        ];
      };
    };
}
```

## Verification

Test the devShell:

```bash
# Check flake evaluates
nix flake check

# Enter devShell
nix develop

# Test fish is running
fish --version

# Test abbreviations
abbr -a

# Test with command (verifies interactive check works)
nix develop --command echo "Success"
```
